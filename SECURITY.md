# Security Policy

## Supported Versions

Since FLUX is a proprietary codebase, security updates are strictly managed and backported to active release branches under development. Only the latest release branches receive security updates.

| Version | Supported |
| :--- | :---: |
| 2.x | Yes |
| 1.x | No |
| < 1.0 | No |

---

## Reporting a Vulnerability

We take the security of our codebase seriously. If you find a security vulnerability, please do not disclose it publicly or open a public issue on GitHub. Doing so exposes the user base to unnecessary security risk before a fix is available.

Instead, please report security vulnerabilities directly to the project owner by contacting Sachin Sharma. 

### Reporting Guidelines
1. **Details:** Provide a detailed description of the vulnerability, including step-by-step instructions to reproduce it.
2. **Impact:** Detail the security impact (e.g., local privilege escalation, memory leakage, buffer corruption).
3. **Environment:** Include information about the environment, target device profile, and API level used during reproduction.

All valid vulnerability reports will be investigated promptly. Once resolved, security patches will be committed directly to active version branches. We appreciate your assistance in keeping the FLUX project secure.
